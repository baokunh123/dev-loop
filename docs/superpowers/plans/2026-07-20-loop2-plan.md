# Loop 2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a polling server and trigger script that auto-picks up approved plans from `ready/`, runs Codex full-auto to implement → verify → open PR, with one retry and escalation to `failed/` on exhaustion.

**Architecture:** `loop2-server.js` polls `ready/` on a cron schedule and spawns `loop-2-trigger.sh <ticket>` per discovered ticket in parallel. The bash script acquires a lockfile, runs `codex --approval-mode full-auto`, reads `result.json` written by Codex, and moves the ticket to `done/` or `failed/` accordingly.

**Tech Stack:** Node.js 18+, node-cron, child_process.spawn, bash, codex CLI

## Global Constraints

- Node.js >= 18
- Weekdays 9-5 PST only: cron `0 9-17 * * 1-5`, timezone `America/Los_Angeles`
- All operations logged to stdout with ISO timestamp
- Never crash on errors — log and continue
- Lockfile stale threshold: 2 hours (same as Loop 1)
- Codex must be invoked with `--approval-mode full-auto`
- `done/` and `failed/` directories must exist before server starts

```bash
#!/bin/bash
set -euo pipefail

TICKET="${1:?Usage: loop-2-trigger.sh <ticket>}"
BASE=~/workspace/dev-loop
READY="$BASE/ready/$TICKET"
LOCKFILE="$BASE/ready/$TICKET.lock"
NOW=$(date -u +%Y-%m-%dT%H:%M:%SZ)
SESSION="${NOW//[^0-9]/}"
SESSION="${SESSION: -8}"

log() { echo "[$(date -u +%Y-%m-%dT%H:%M:%SZ)] [loop2] [$TICKET] $*"; }

# Lockfile check
if [[ -f "$LOCKFILE" ]]; then
  ACQUIRED=$(python3 -c "import json; print(json.load(open('$LOCKFILE'))['acquired'])")
  ACQUIRED_S=$(date -j -f "%Y-%m-%dT%H:%M:%SZ" "$ACQUIRED" +%s 2>/dev/null)
  NOW_S=$(date +%s)
  AGE_H=$(( (NOW_S - ACQUIRED_S) / 3600 ))
  if [[ $AGE_H -lt 2 ]]; then
    log "Skipped — lockfile active (age: ${AGE_H}h)"
    exit 0
  fi
  log "Stale lockfile detected (age: ${AGE_H}h), overwriting"
fi

echo "{\"session\": \"$SESSION\", \"acquired\": \"$NOW\"}" > "$LOCKFILE"
log "Lockfile acquired"

CODEX_PROMPT="Read ~/workspace/dev-loop/ready/$TICKET/plan.md.
Use superpowers:subagent-driven-development to implement all tasks.
When complete, run superpowers:verification-before-completion.
Then run superpowers:finishing-a-development-branch.
Work in ~/workspace/mortgage-graphify.
When done, write ~/workspace/dev-loop/ready/$TICKET/result.json:
- On success: {\"status\": \"success\", \"pr_url\": \"<url>\"}
- On failure: {\"status\": \"failed\", \"reason\": \"<reason>\"}"

run_codex() {
  local attempt=$1
  log "Starting Codex attempt $attempt"
  rm -f "$READY/result.json"
  codex exec "$CODEX_PROMPT" || true
}

read_result() {
  if [[ ! -f "$READY/result.json" ]]; then
    echo "result.json missing"
    return 1
  fi
  python3 -c "import json,sys; d=json.load(open('$READY/result.json')); sys.exit(0 if d['status']=='success' else 1)" 2>/dev/null
}

read_reason() {
  if [[ ! -f "$READY/result.json" ]]; then
    echo "result.json missing"
  else
    python3 -c "import json; d=json.load(open('$READY/result.json')); print(d.get('reason', d.get('pr_url', 'unknown')))"
  fi
}

# Attempt 1
run_codex 1
REASON1=$(read_reason)
if read_result; then
  log "Attempt 1 succeeded — PR: $REASON1"
  mv "$READY" "$BASE/done/$TICKET"
  rm -f "$LOCKFILE"
  log "Moved to done/"
  exit 0
fi
log "Attempt 1 failed: $REASON1"

# Attempt 2 (retry)
run_codex 2
REASON2=$(read_reason)
if read_result; then
  log "Attempt 2 succeeded — PR: $REASON2"
  mv "$READY" "$BASE/done/$TICKET"
  rm -f "$LOCKFILE"
  log "Moved to done/"
  exit 0
fi
log "Attempt 2 failed: $REASON2"

# Both failed — escalate
mkdir -p "$BASE/failed/$TICKET"
cat > "$BASE/failed/$TICKET/error.md" <<EOF
# $TICKET — Failed after 2 attempts

Attempt 1: $REASON1
Attempt 2: $REASON2
Timestamp: $NOW
EOF
mv "$READY"/* "$BASE/failed/$TICKET/" 2>/dev/null || true
rmdir "$READY" 2>/dev/null || true
rm -f "$LOCKFILE"
log "Escalated to failed/"
exit 1
```

### Task 3: Implement `loop2-server.js`

**Files:**
- Create: `~/workspace/dev-loop/loop2-server.js`

**Interfaces:**
- Consumes: `~/workspace/dev-loop/ready/` directory
- Consumes: `~/workspace/dev-loop/ready/<ticket>.lock` (lockfile check before spawn)
- Produces: spawned `loop-2-trigger.sh` processes per ticket

- [ ] **Step 1: Write `loop2-server.js`**

```js
import cron from 'node-cron'
import fs from 'fs'
import path from 'path'
import { spawn } from 'child_process'
import { fileURLToPath } from 'url'
import 'dotenv/config'

const __dirname = path.dirname(fileURLToPath(import.meta.url))
const READY_DIR = path.join(__dirname, 'ready')
const TRIGGER = path.join(process.env.HOME, '.claude/skills/loop-2-trigger/loop-2-trigger.sh')

function log(msg) {
  console.log(`[${new Date().toISOString()}] [loop2] ${msg}`)
}

function isLockActive(ticket) {
  const lockfile = path.join(READY_DIR, `${ticket}.lock`)
  if (!fs.existsSync(lockfile)) return false
  try {
    const { acquired } = JSON.parse(fs.readFileSync(lockfile, 'utf8'))
    const ageH = (Date.now() - new Date(acquired).getTime()) / 3_600_000
    return ageH < 2
  } catch {
    return false
  }
}

function findPendingTickets() {
  if (!fs.existsSync(READY_DIR)) return []
  return fs.readdirSync(READY_DIR, { withFileTypes: true })
    .filter(d => d.isDirectory() && fs.existsSync(path.join(READY_DIR, d.name, 'plan.md')))
    .map(d => d.name)
}

function spawnTrigger(ticket) {
  log(`Spawning trigger for ${ticket}`)
  const proc = spawn('bash', [TRIGGER, ticket], {
    detached: true,
    stdio: 'ignore',
  })
  proc.on('error', err => log(`Spawn error for ${ticket}: ${err.message}`))
  proc.unref()
}

function poll() {
  log('Poll start')
  const tickets = findPendingTickets()
  log(`Tickets found: ${tickets.length}`)

  for (const ticket of tickets) {
    if (isLockActive(ticket)) {
      log(`Skipped (lockfile active): ${ticket}`)
      continue
    }
    spawnTrigger(ticket)
  }

  log('Poll end')
}

// weekdays 9-5 PST, every hour on the hour
cron.schedule('0 9-17 * * 1-5', poll, { timezone: 'America/Los_Angeles' })
log('loop2-server.js started — polling weekdays 9-17 PST')

// run immediately on start
poll()
```

```markdown
# Loop 2 Trigger

Loop 2 is triggered by `loop2-server.js` via bash, not by Claude Code CronCreate.

This skill is documentation only — it records the Codex prompt and expected behavior.

## What `loop-2-trigger.sh` does

1. Acquires a lockfile (`ready/<ticket>.lock`, 2-hour stale threshold)
2. Runs `codex --approval-mode full-auto` with the prompt below
3. Reads `ready/<ticket>/result.json` written by Codex
4. Success → moves ticket to `done/<ticket>/`
5. Failure → retries once; second failure → moves to `failed/<ticket>/` with `error.md`

## Codex prompt

```
Read ~/workspace/dev-loop/ready/<ticket>/plan.md.
Use superpowers:subagent-driven-development to implement all tasks.

Then run superpowers:finishing-a-development-branch.
Work in ~/workspace/mortgage-graphify.
When done, write ~/workspace/dev-loop/ready/<ticket>/result.json:
- On success: {"status": "success", "pr_url": "<url>"}
- On failure: {"status": "failed", "reason": "<reason>"}
```

## Human gate

- **Success**: PR opened on GitHub — review and merge normally.
- **Failure**: Ticket appears in `failed/` — read `error.md`, fix plan or ticket, move back to `ready/` to retry.
```
