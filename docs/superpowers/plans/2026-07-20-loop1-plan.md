# Loop 1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a polling server that pulls Jira tickets to local disk, and a Claude Code skill that picks them up and runs brainstorming to produce implementation plans.

**Architecture:** `server.js` runs on a node-cron schedule, queries Jira for tickets with `claude-ready` label, and writes raw ticket content to `clarified/<ticket>.md`. A `loop-1-triage` Claude Code skill runs on CronCreate, scans `clarified/` for unprocessed tickets, acquires a lockfile, and triggers `superpowers:brainstorming` → `superpowers:writing-plans`.

**Tech Stack:** Node.js, node-cron, dotenv, node-fetch (or native fetch Node 18+)

## Global Constraints

- Node.js >= 18 (native fetch available)
- Weekdays 9-5 PST only for all scheduling: cron expression `0 9-17 * * 1-5`
- All operations logged to stdout with ISO timestamp
- Never crash on Jira API errors — log and continue
- All specs and plans produced by brainstorming must be written in both English (`.md`) and Chinese (`.zh.md`)
- Lockfile stale threshold: 2 hours

### Task 2: Implement `server.js` — Jira poller

**Files:**
- Create: `~/workspace/dev-loop/server.js`

**Interfaces:**
- Consumes: `JIRA_BASE`, `JIRA_EMAIL`, `JIRA_TOKEN` from `.env`; `processed.json` for dedup state
- Produces: `clarified/<ticket-key>.md` per discovered ticket

- [ ] **Step 1: Write `server.js`**

```js
import cron from 'node-cron'
import fs from 'fs'
import path from 'path'
import { fileURLToPath } from 'url'
import 'dotenv/config'

const __dirname = path.dirname(fileURLToPath(import.meta.url))
const PROCESSED_FILE = path.join(__dirname, 'processed.json')
const CLARIFIED_DIR = path.join(__dirname, 'clarified')

function log(msg) {
  console.log(`[${new Date().toISOString()}] ${msg}`)
}

function loadProcessed() {
  try { return JSON.parse(fs.readFileSync(PROCESSED_FILE, 'utf8')) }
  catch { return {} }
}

function markProcessed(key) {
  const state = loadProcessed()
  state[key] = new Date().toISOString()
  fs.writeFileSync(PROCESSED_FILE, JSON.stringify(state, null, 2))
}

function isProcessed(key) {
  return !!loadProcessed()[key]
}

async function fetchTickets() {
  const jql = 'assignee = currentUser() AND status = "In Progress" AND labels = "claude-ready"'
  const url = `${process.env.JIRA_BASE}/rest/api/3/search?jql=${encodeURIComponent(jql)}&expand=renderedFields&fields=summary,description,comment,updated`
  const auth = Buffer.from(`${process.env.JIRA_EMAIL}:${process.env.JIRA_TOKEN}`).toString('base64')

  const res = await fetch(url, {
    headers: { Authorization: `Basic ${auth}`, Accept: 'application/json' }
  })

  if (!res.ok) throw new Error(`Jira API ${res.status}: ${await res.text()}`)
  return (await res.json()).issues ?? []
}

function writeTicket(issue) {
  const dir = CLARIFIED_DIR
  const filePath = path.join(dir, `${issue.key}.md`)
  const comments = (issue.fields.comment?.comments ?? [])
    .map(c => `**${c.author.displayName}** (${c.created}):\n${c.body}`)
    .join('\n\n---\n\n')

  const content = `# ${issue.key}: ${issue.fields.summary}

**Fetched:** ${new Date().toISOString()}

## Description

${issue.fields.description ?? '_No description_'}

## Comments

${comments || '_No comments_'}
`
  fs.writeFileSync(filePath, content)
  log(`Written: ${filePath}`)
}

async function poll() {
  log('Poll start')
  let issues
  try {
    issues = await fetchTickets()
  } catch (err) {
    log(`Jira API error: ${err.message}`)
    log('Poll end (error)')
    return
  }

  log(`Tickets found: ${issues.length}`)

  for (const issue of issues) {
    const dedupeKey = `${issue.key}-${issue.fields.updated}`
    if (isProcessed(dedupeKey)) {
      log(`Skipped (dedup): ${issue.key}`)
      continue
    }
    writeTicket(issue)
    markProcessed(dedupeKey)
  }

  log('Poll end')
}

// weekdays 9-5 PST, every hour on the hour
cron.schedule('0 9-17 * * 1-5', poll, { timezone: 'America/Los_Angeles' })
log('server.js started — polling weekdays 9-17 PST')

// run immediately on start
poll()
```

- [ ] **Step 2: Write `SKILL.md`**

```markdown
---
name: loop-1-triage
description: Scan clarified/ for unprocessed Jira tickets and run brainstorming to produce plans
user_invocable: false
---

# Loop 1 Triage Skill

You are the Loop 1 triage agent. Run this skill on a CronCreate schedule (`0 9-17 * * 1-5` PST).

## Step 1: Scan for pending tickets

Look in `~/workspace/dev-loop/clarified/` for files matching `*.md` (not inside subdirectories) that do NOT have a corresponding `<ticket>/plan.md`.

Example: `FPP-1234.md` exists but `FPP-1234/plan.md` does not → pending.

If no pending tickets found → exit silently. Do not log anything.

## Step 2: Pick one ticket

Select the ticket with the lowest number (e.g. FPP-1234 before FPP-1235).

## Step 3: Acquire lockfile

Check for `~/workspace/dev-loop/clarified/<ticket>.lock`.

- **Not present** → write the lockfile:
  ```
  {"session": "<random 8-char hex>", "acquired": "<ISO timestamp>"}
  ```
- **Present, timestamp < 2 hours ago** → another session is processing this ticket. Exit silently.
- **Present, timestamp >= 2 hours ago** → stale lock. Overwrite it and proceed.

## Step 4: Read ticket content

Read `~/workspace/dev-loop/clarified/<ticket>.md` in full.

## Step 5: Run brainstorming

Invoke `superpowers:brainstorming` with the ticket content as context. The goal is to produce a clear implementation plan.

Pass this context to brainstorming:
> "This is a Jira ticket for the dev-loop repo. Please run brainstorming to clarify requirements and produce an implementation plan. Ticket content: [paste full ticket content]"

## Step 6: Run writing-plans

After brainstorming completes, invoke `superpowers:writing-plans` to write the plan.

Save output to:
- `~/workspace/dev-loop/clarified/<ticket>/plan.md` (English)
- `~/workspace/dev-loop/clarified/<ticket>/plan.zh.md` (Chinese)

## Step 7: Release lockfile

Delete `~/workspace/dev-loop/clarified/<ticket>.lock`.

## Step 8: Done

The ticket folder `clarified/<ticket>/` now contains `plan.md` and `plan.zh.md`. The human will review and move it to `ready/<ticket>/` to approve for Loop 2.
```
