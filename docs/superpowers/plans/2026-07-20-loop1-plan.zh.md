# Loop 1 实施计划

> **给 AI 执行者：** 必须使用 superpowers:subagent-driven-development（推荐）或 superpowers:executing-plans 逐任务执行本计划。步骤使用复选框（`- [ ]`）语法追踪进度。

**目标：** 构建一个轮询服务器将 Jira ticket 拉取到本地磁盘，以及一个 Claude Code skill 负责 pick up ticket 并运行 brainstorming 产出实施计划。

**架构：** `server.js` 按 node-cron 计划运行，查询带有 `claude-ready` 标签的 Jira ticket，将原始内容写入 `clarified/<ticket>.md`。`loop-1-triage` Claude Code skill 通过 CronCreate 触发，扫描 `clarified/` 中未处理的 ticket，获取 lockfile，然后触发 `superpowers:brainstorming` → `superpowers:writing-plans`。

**技术栈：** Node.js、node-cron、dotenv、原生 fetch（Node 18+）

## 全局约束

- Node.js >= 18（原生 fetch 可用）
- 所有调度仅限工作日 9-5 PST：cron 表达式 `0 9-17 * * 1-5`
- 所有操作记录到 stdout，带 ISO 时间戳
- Jira API 错误时不崩溃——记录日志并继续
- brainstorming 产出的所有 spec 和 plan 必须同时输出英文（`.md`）和中文（`.zh.md`）两份
- lockfile 过期阈值：2 小时

### Task 2：实现 `server.js` — Jira 轮询器

**文件：**
- 新建：`~/workspace/dev-loop/server.js`

**接口：**
- 消费：`.env` 中的 `JIRA_BASE`、`JIRA_EMAIL`、`JIRA_TOKEN`；`processed.json` 用于去重状态
- 产出：每个发现的 ticket 对应 `clarified/<ticket-key>.md`

- [ ] **步骤 1：编写 `server.js`**

```js
import cron from 'node-cron'
import fs from 'fs'
import path from 'path'
import { fileURLToPath } from 'url'
import 'dotenv/config'

const __dirname = path.dirname(fileURLToPath(import.meta.url))
const PROCESSED_FILE = path.join(__dirname, 'processed.json')
const CLARIFIED_DIR = path.join(__dirname, 'clarified')

function log(msg) {
  console.log(`[${new Date().toISOString()}] ${msg}`)
}

function loadProcessed() {
  try { return JSON.parse(fs.readFileSync(PROCESSED_FILE, 'utf8')) }
  catch { return {} }
}

function markProcessed(key) {
  const state = loadProcessed()
  state[key] = new Date().toISOString()
  fs.writeFileSync(PROCESSED_FILE, JSON.stringify(state, null, 2))
}

function isProcessed(key) {
  return !!loadProcessed()[key]
}

async function fetchTickets() {
  const jql = 'assignee = currentUser() AND status = "In Progress" AND labels = "claude-ready"'
  const url = `${process.env.JIRA_BASE}/rest/api/3/search?jql=${encodeURIComponent(jql)}&expand=renderedFields&fields=summary,description,comment,updated`
  const auth = Buffer.from(`${process.env.JIRA_EMAIL}:${process.env.JIRA_TOKEN}`).toString('base64')

  const res = await fetch(url, {
    headers: { Authorization: `Basic ${auth}`, Accept: 'application/json' }
  })

  if (!res.ok) throw new Error(`Jira API ${res.status}: ${await res.text()}`)
  return (await res.json()).issues ?? []
}

function writeTicket(issue) {
  const filePath = path.join(CLARIFIED_DIR, `${issue.key}.md`)
  const comments = (issue.fields.comment?.comments ?? [])
    .map(c => `**${c.author.displayName}** (${c.created}):\n${c.body}`)
    .join('\n\n---\n\n')

  const content = `# ${issue.key}: ${issue.fields.summary}

**拉取时间：** ${new Date().toISOString()}

## 描述

${issue.fields.description ?? '_无描述_'}

## 评论

${comments || '_无评论_'}
`
  fs.writeFileSync(filePath, content)
  log(`Written: ${filePath}`)
}

async function poll() {
  log('Poll start')
  let issues
  try {
    issues = await fetchTickets()
  } catch (err) {
    log(`Jira API error: ${err.message}`)
    log('Poll end (error)')
    return
  }

  log(`Tickets found: ${issues.length}`)

  for (const issue of issues) {
    const dedupeKey = `${issue.key}-${issue.fields.updated}`
    if (isProcessed(dedupeKey)) {
      log(`Skipped (dedup): ${issue.key}`)
      continue
    }
    writeTicket(issue)
    markProcessed(dedupeKey)
  }

  log('Poll end')
}

// 工作日 9-5 PST，每小时整点
cron.schedule('0 9-17 * * 1-5', poll, { timezone: 'America/Los_Angeles' })
log('server.js started — polling weekdays 9-17 PST')

// 启动时立即执行一次
poll()
```

```markdown
---
name: loop-1-triage
description: 扫描 clarified/ 中未处理的 Jira ticket 并运行 brainstorming 产出计划
user_invocable: false
---

# Loop 1 Triage Skill

你是 Loop 1 的 triage agent。本 skill 通过 CronCreate 调度（`0 9-17 * * 1-5` PST）运行。

## 步骤 1：扫描待处理 ticket

在 `~/workspace/dev-loop/clarified/` 中查找顶层 `*.md` 文件（不在子目录中），且没有对应 `<ticket>/plan.md` 的。

示例：`FPP-1234.md` 存在但 `FPP-1234/plan.md` 不存在 → 待处理。

如果没有待处理 ticket → 静默退出，不做任何操作。

## 步骤 2：选取一个 ticket

选取编号最小的 ticket（如 FPP-1234 优先于 FPP-1235）。

## 步骤 3：获取 lockfile

检查 `~/workspace/dev-loop/clarified/<ticket>.lock` 是否存在。

- **不存在** → 写入 lockfile：
  ```
  {"session": "<随机 8 位十六进制>", "acquired": "<ISO 时间戳>"}
  ```
- **存在，时间戳 < 2 小时前** → 另一个 session 正在处理此 ticket，静默退出。
- **存在，时间戳 >= 2 小时前** → 视为过期 lock，覆盖并继续。

## 步骤 4：读取 ticket 内容

完整读取 `~/workspace/dev-loop/clarified/<ticket>.md`。

## 步骤 5：运行 brainstorming

携带 ticket 内容调用 `superpowers:brainstorming`，目标是产出清晰的实施计划。

传递给 brainstorming 的上下文：
> "这是 dev-loop repo 的一个 Jira ticket。请运行 brainstorming 澄清需求并产出实施计划。Ticket 内容：[粘贴完整 ticket 内容]"

## 步骤 6：运行 writing-plans

brainstorming 完成后，调用 `superpowers:writing-plans` 编写计划。

输出保存到：
- `~/workspace/dev-loop/clarified/<ticket>/plan.md`（英文）
- `~/workspace/dev-loop/clarified/<ticket>/plan.zh.md`（中文）

## 步骤 7：释放 lockfile

删除 `~/workspace/dev-loop/clarified/<ticket>.lock`。

## 步骤 8：完成

ticket 文件夹 `clarified/<ticket>/` 现在包含 `plan.md` 和 `plan.zh.md`。
你审阅后将其移动到 `ready/<ticket>/` 即为批准，交给 Loop 2。
```
